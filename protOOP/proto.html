Объекты в JavaScript можно организовать в цепочки таким образом, что если искомое свойство не найденное в одном объекте, будет искаться в другом. Связующим звяном выступает свойство __proto__ Если один объект имеет ссылку __proto__ на другой объект, то при чтении свойства из него, если свойство отсутствует в самом объекте, оно ищется в объекте __proto__. Одному объекту нельзя присвоить несколько прототипов через __proto__, необходимо образовавыть цепочки Объект, на который указывается ссылка __proto__ называется "прототипом". Прототип используется только для чтения Прототип это хранилище свойств и методов объекта, автоматически используемое в поиске Цикл for..in пробигается и по прототипам объекта Object.keys() - не заходит в прототип У бъекта есть наследуемые свойства, чтобы избежать их можно создать объект без них Объект, создаваемый при помощи Object.create(null) не имеет прототипа, а значит в нём нет лишних свойств. Для коллекции – как раз то, что надо. Несмотря на то, в каком прототипе находится свойство, это никак не влияет на установку this, которая осуществляется по своим, независимым правилам. Не стоит оптимизировать поиск через цепочки, так как нет разницы, брать свойство из объекта или прототипа. Они запоминают, где было найдено свойство и в следующий раз при запросе, к примеру, pockets.glasses начнут искать сразу в прототипе (head).
